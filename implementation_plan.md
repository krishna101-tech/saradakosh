# Saradakosh Web Application Plan

Create a full-fledged, mobile-friendly web application for Saradakosh that handles large datasets instantaneously, features a themed aesthetic, and provides robust interactive components including "Today in History" and drill-down reports.

## Architecture & Tech Stack

Your suggestion is absolutely brilliant and follows the modern "Jamstack" architecture. By splitting the application into two parts, we achieve **maximum speed on the internet** while giving you a **powerful offline editing tool**.

### Part 1: The Local Admin Dashboard (Offline CMS)
- **Framework**: Next.js (React) running on your local machine (`localhost`).
- **Database**: SQLite. This provides the robust relational database needed to replicate your MS Access forms, handle dropdowns, and save edits securely.
- **Publish Feature**: A "Publish to Web" button that automatically extracts all the SQLite data, compiles it into highly optimized JSON files, and builds the public website.

### Part 2: The Public Website (Internet Facing)
- **Framework**: Static React (via Vite) or Static Next.js.
- **Data Source**: Pre-compiled Static JSON files generated by your Admin Dashboard.
- **Speed**: Truly **instantaneous**. Since it's just loading static JSON files into memory, there is zero database latency. It can be hosted anywhere for free (GitHub Pages, Vercel) with no backend to hack or maintain.

## Design Aesthetics
To beautifully evoke the era of Sri Ramakrishna, Ma Sarada, and Swami Vivekananda:
- **Colors**: Saffron/Ochre accents, warm parchment/off-white backgrounds, deep earthy browns and charcoal for text.
- **Typography**: Classic, readable Serif fonts (like *Merriweather* or *Playfair Display*) for headings to give a book-like historical feel, paired with a clean Sans-Serif (like *Inter* or *Lora*) for dense reading text.
- **UI Elements**: Soft shadows, subtle paper textures, and elegant dividers.

## Proposed Implementation Steps

### 1. Database Migration & Local Admin Setup
- Create `import_to_sqlite.py` to migrate your Excel tables into a clean SQLite database (`saradakosh.db`).
- **Zero-Loss Data Verification**: I will write a strict `verify_migration.py` script. This script will query the new SQLite database, dynamically reconstruct the data back into your old MS Access table formats (flattening the new tables back into M1, M2 columns, etc.), and run a row-by-row `pandas` mathematical comparison against your original Excel files. **I will not proceed with building the app until this script outputs a 100% identical match, guaranteeing zero data loss or broken relationships.**
- Build the **Local Admin Dashboard** in Next.js.
- Create the "Record Editor" form (mirroring your MS Access screenshot) with saving, duplicating, deleting, and dropdown logic.

### 2. The "Publish" Pipeline
- Write a Node.js/Python script that triggers when you click "Publish".
- This script pulls all data from SQLite and generates `du1_data.json`, `matrix.json`, etc.

### 3. Public Website Features (Static JSON driven)
- **Home Page**:
  - Hero section with Title and Subtitle.
  - **Today in History**: Instantly filters the in-memory JSON data for today's Day and Month, grouping by Year. Level 1 will be the `DU` text. Click to expand Level 2 (related records/children).
  - **Reports Directory**: A visual list of all generated reports.
- **Public Report Views**: Clickable, drill-down chronological lists that load instantaneously from JSON.
